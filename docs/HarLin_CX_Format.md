# HarLin CX 可执行文件格式

## 1. 概述

`.cx` 文件格式是 HarLin 内核的原生可执行文件格式。它为无需外部依赖即可简单解析和加载。

## 2. 文件布局

| 偏移 | 大小 | 字段 |
|------|------|------|
| 0x00 | 8 | 魔数：`HARLINCX` |
| 0x08 | 2 | 版本号（当前为 1） |
| 0x0A | 2 | 标志位 |
| 0x0C | 4 | 入口点偏移（相对于代码段起点） |
| 0x10 | 8 | 代码段在文件中的偏移 |
| 0x18 | 8 | 代码段大小（字节） |
| 0x20 | 8 | 数据段在文件中的偏移 |
| 0x28 | 8 | 数据段大小（字节） |
| 0x30 | 8 | BSS 段大小（字节） |
| 0x38 | 8 | 重定位表偏移 |
| 0x40 | 8 | 重定位项数量 |
| 0x48 | 8 | 栈大小（字节） |
| 0x50 | - | 头部填充至 0x200 字节 |

## 3. 段加载

- 代码段加载到用户态可执行页面。
- 数据段加载到用户态可写页面。
- BSS 段在数据段之后被清零初始化。
- 所有段在内存中按页对齐。

## 4. 重定位

每个重定位项为 8 字节，包含一个相对于代码段起点的偏移量，该位置需要被修补为代码段运行时加载地址加上存储的加数。

| 偏移 | 大小 | 字段 |
|------|------|------|
| 0x00 | 8 | 待修补偏移 |

加载器读取 `code_base + offset` 处的 64 位值，加上运行时代码基地址，然后将结果写回。

## 5. 入口点

执行从 `code_base + entry_offset` 开始。加载器设置以下寄存器：

- `RSP`：分配的用户栈顶部。
- `RDI`：参数块指针，如果没有参数则为 0。

## 6. .cx 程序 ABI

- 程序在 64 位长模式的 ring 3 下运行。
- 使用 System V AMD64 调用约定。
- 系统调用通过中断 0x80 发起，调用号放在 `RAX` 中。
- 栈向下增长，调用前必须 16 字节对齐。

## 7. 工具链

`.cx` 文件由 HarLin 工具链从编译后的目标文件生成。工具链负责生成头部、段、重定位表和系统调用桩。
