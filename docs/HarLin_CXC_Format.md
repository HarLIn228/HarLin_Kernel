# HarLin CXC 可执行文件格式

## 1. 概述

`.cxc` 文件格式是 HarLin 内核的原生可执行文件格式。它为无需外部依赖即可简单解析和加载。

CXC（Compact eXecutable Core）更开放的运行格式

## 2. 文件布局

| 偏移 | 大小 | 字段 |
|------|------|------|
| 0x00 | 9 | 魔数：`HARLINCXC` |
| 0x09 | 7 | 保留（必须为零） |
| 0x10 | 2 | 版本号（当前为 1） |
| 0x12 | 2 | 标志位 |
| 0x14 | 4 | 入口点偏移（相对于代码段起点） |
| 0x18 | 8 | 代码段在文件中的偏移 |
| 0x20 | 8 | 代码段大小（字节） |
| 0x28 | 8 | 数据段在文件中的偏移 |
| 0x30 | 8 | 数据段大小（字节） |
| 0x38 | 8 | BSS 段大小（字节） |
| 0x40 | 8 | 重定位表偏移 |
| 0x48 | 8 | 重定位项数量 |
| 0x50 | 8 | 栈大小（字节） |
| 0x58 | - | 头部填充至 0x200 字节 |

## 3. 段加载

- 代码段加载到用户态可执行页面。
- 数据段加载到用户态可写页面。
- BSS 段在数据段之后被清零初始化。
- 所有段在内存中按页对齐。
- 代码段大小必须大于 0，且入口点必须位于代码段范围内。

## 4. 重定位

每个重定位项为 8 字节，包含一个相对于代码段起点的偏移量，该位置需要被修补为代码段运行时加载地址加上存储的加数。

| 偏移 | 大小 | 字段 |
|------|------|------|
| 0x00 | 8 | 待修补偏移 |

加载器读取 `code_base + offset` 处的 64 位值，加上运行时代码基地址，然后将结果写回。

加载器会拒绝未 8 字节对齐或超出代码段范围的重定位项。

## 5. 入口点

执行从 `code_base + entry_offset` 开始。加载器设置以下寄存器：

- `RSP`：分配的用户栈顶部。
- `RDI`：参数块指针，如果没有参数则为 0。

## 6. .cxc 程序 ABI

- 程序在 64 位长模式的 ring 3 下运行。
- 使用 System V AMD64 调用约定。
- 系统调用通过中断 0x80 发起，调用号放在 `RAX` 中。
- 栈向下增长，调用前必须 16 字节对齐。

## 7. 健壮性保证

CXC 加载器在加载过程中执行以下检查：

- 文件大小至少为 `CXC_HEADER_SIZE`（0x200 字节）。
- 魔数必须完全匹配 `HARLINCXC`。
- 版本号必须为 1。
- 代码段、数据段、BSS、栈大小均不能超过 `USER_MAX_SIZE`（1MB）。
- 各段偏移不能小于头部大小，不能超出文件范围。
- 数据段、BSS、栈的内存布局不会发生回绕。
- 重定位偏移必须 8 字节对齐，且位于代码段范围内。
- 任何映射失败都会回滚已分配的页面，避免内存泄漏。

## 8. 工具链

`.cxc` 文件由 HarLin 工具链从编译后的目标文件生成。工具链负责生成头部、段、重定位表和系统调用桩。
