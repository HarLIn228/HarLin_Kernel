# 第三方驱动接入教程

本文面向想要为 HarLin Kernel 编写第三方驱动的开发者，演示从零开始接入一个自定义硬件驱动的完整流程。

## 一、驱动模型

HarLin 的驱动是「**注册 ops**」式：驱动实现一组 `xxx_ops` 接口，然后调用 `drv_register()` 把自己挂到内核的驱动总线上。内核在合适的时候（启动时 `drv_load_all`）调用 `probe()` 探测硬件、`init()` 初始化设备。

### 1.1 子系统分类

### 1.2 驱动 ops 结构

```c
struct drv_ops {
    const char* name;                    // 驱动名（必须唯一）
    enum drv_subsys subsys;              // 归属子系统
    int  (*probe)(void* ctx);            // 探测硬件是否在位
    int  (*init)(void* ctx);             // 初始化硬件
    void (*remove)(void* ctx);           // 卸载时清理
    int  (*suspend)(void* ctx);          // 休眠（可选）
    int  (*resume)(void* ctx);           // 恢复（可选）
    int  priority;                       // 优先级，越大越优先
};
```

### 1.3 Provider 探测器

如果你的驱动只能运行在特定硬件上（例如 NVMe SSD），可以先注册一个 `provider`：

```c
struct drv_provider {
    const char* name;
    int  (*scan)(void);                  // 扫描硬件是否在位
    int  available;                      // 由 drv_loader 填写
};
```

`drv_provider_scan_all()` 会调用所有 `scan()`，把结果写入 `available`，驱动 `init()` 时可检查 `drv_provider_available("xxx")`。

## 二、完整示例：编写一个虚拟网卡驱动

下面演示一个「loopback 虚拟网卡」驱动的完整流程。当真实 NIC 驱动都不可用时，内核回退到 loopback，保证网络子系统仍可工作。

### 2.1 创建源文件

新建 `src/harlin/drv/net_loopback.c`：

```c
#include "drv_loader.h"
#include "harlin_API.h"

static int lb_probe(void* ctx)
{
    (void)ctx;
    return 0;  // loopback 永远在位
}

static int lb_init(void* ctx)
{
    (void)ctx;
    Harlin_Print("[net_loopback] registered\n");
    return 0;
}

static void lb_remove(void* ctx) { (void)ctx; }

static struct drv_ops lb_ops = {
    .name = "net_loopback",
    .subsys = DRV_SUBSYS_NET,
    .probe = lb_probe,
    .init = lb_init,
    .remove = lb_remove,
    .priority = -100,  // 优先级最低，仅作为回退
};

void drv_net_loopback_init(void)
{
    drv_register(&lb_ops, 0);
}
```

### 2.2 在启动流程注册

编辑 `src/harlin/syscall/harlin_API.c`，在 `drv_load_all()` 之前调用你的初始化函数：

```c
#include "net_loopback.h"  // 你的头

void Harlin_Boot(void)
{
    // ... 既有代码 ...
    drv_loader_init();
    drv_net_loopback_init();  // 注册
    drv_load_all();           // 实际探测+初始化
    // ...
}
```

### 2.3 添加头路径

如果你的头文件放在新目录，需要在 `build.bat` 的 `CFLAGS` 中加入 `-I src\head\新目录`。

## 三、实战：接入 NVMe 驱动骨架

### 3.1 创建 `src/harlin/drv/nvme.c`

```c
#include "drv_loader.h"
#include "pci.h"
#include "harlin_API.h"

static int nvme_probe(void* ctx)
{
    (void)ctx;
    return pci_find_class(0x01, 0x08) >= 0 ? 0 : -1;  // 01/08 = NVMe
}

static int nvme_init(void* ctx)
{
    (void)ctx;
    Harlin_Print("[nvme] init stub - TODO\n");
    return 0;
}

static void nvme_remove(void* ctx) { (void)ctx; }

static struct drv_ops nvme_ops = {
    .name = "drv_nvme",
    .subsys = DRV_SUBSYS_STORAGE,
    .probe = nvme_probe,
    .init = nvme_init,
    .remove = nvme_remove,
    .priority = 100,
};

void drv_nvme_init(void)
{
    drv_register(&nvme_ops, 0);
}
```

### 3.2 编译

运行 `.\build.bat`，新文件会被自动包含（`build.bat` 已使用 `for /r` 递归遍历 `src\harlin\*.c`）。

### 3.3 启动验证

启动后日志应出现：

```
[BOOT] drv loaded
[drv_nvme] init stub - TODO
```

如果 PCI 总线没扫描到 NVMe 设备，`probe()` 返回非 0，驱动被跳过，不影响其他驱动。

## 四、调试技巧

- `drv_active_count(DRV_SUBSYS_NET)` 打印当前活跃的网卡数量
- `drv_first_active(DRV_SUBSYS_STORAGE)` 返回当前激活的存储驱动名
- `drv_provider_available("nvme_pci")` 检查某个 provider 是否在位
- 注册失败时 `drv_register` 返回 -1，注意 `DRV_PROVIDERS_MAX` 上限

## 五、最佳实践

1. **probe() 必须轻量**：只检查硬件 ID、PCI BAR，不做 MMIO
2. **init() 才做实质初始化**：映射 BAR、注册中断、初始化数据结构
3. **remove() 顺序对称**：释放 init 中分配的资源
4. **priority 用好**：高优先级抢占低优先级；负数表示回退
5. **不要在 probe/init 中阻塞太久**：影响启动时间
6. **失败必须回滚**：init 中途失败要回退已分配资源
7. **多核安全**：驱动内部状态访问必须加自旋锁

## 七、提交到上游

驱动开发完成后，可通过以下流程贡献到 HarLin Kernel：

1. 在 `src/harlin/drv/` 下创建 `<name>.c` 与对应 `.h`（如有）
2. 在本目录的 `README` 列出驱动能力
3. 提交 PR 到 `github` 远端 `main` 分支
4. PR 描述需包含：硬件 ID 列表、测试环境、限制与已知问题
