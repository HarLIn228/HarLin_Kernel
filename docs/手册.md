# HarLin Kernel 手册

本手册面向第一次接触 HarLin Kernel 的开发者。按照章节顺序操作，即可从零开始构建、修改并运行内核。

## 二、第一次构建与运行

### 2.1 构建

打开 PowerShell，进入项目目录，双击运行 `Build.exe`：

```powershell
cd Kernel
.\Build.exe
```

首次使用请先运行 `Build.exe`，在「工具控制器」中配置各工具路径（支持本地路径或从系统 PATH 自动检测）。配置完成后点击「编译」按钮即可。

如果构建失败，请检查：
- 工具控制器中各工具路径是否正确指向对应的 exe。
- 或使用「工具控制器」中的 Path 模式让程序自动从系统 PATH 中查找。

### 2.2 运行

```powershell
.\run-bochs.bat
```

内核默认纯黑屏启动，不显示任何输出。Bochs 窗口为纯黑屏且无崩溃，即表示内核已成功启动。

如需添加启动画面，请参考第三章。如需查看内核内部初始化过程，可通过 0x0402 调试端口读取 Bochs 日志。

### 2.3 重新构建的习惯

每次修改源码后，都必须重新点击 `Build.exe` 的「编译」按钮，再运行 Bochs。如果运行的是旧镜像，修改不会生效。

### 2.4 构建产物说明

构建完成后，`build/` 目录中的产物：

| 文件 | 大小 | 说明 |
|------|------|------|
| `Binary/boot.bin` | 512 B | 引导扇区，加载到 0x7C00 |
| `Binary/stage2.bin` | ~8 KB | 扩展引导加载器，加载到 0x7E00 |
| `Binary/kernel.bin` | ~107 KB | 内核 ELF 二进制 |
| `boot.img` | — | boot.bin + stage2.bin + kernel.bin 拼接 |
| `HarLin.iso` | ~482 KB | El Torito CD-ROM 镜像 |
| `HarLin.img` | 1.44 MB | 标准软盘镜像 |

## 三、如何添加启动画面

内核默认不显示任何文字。添加启动画面只需要在启动流程中调用显示 API。

### 3.1 打开启动入口

内核启动入口位于 `src/harlin/syscall/harlin_API.c` 的 `Harlin_Boot()` 函数末尾。找到文件末尾附近的闲置循环：

```c
    for (;;) {
        interrupts_enable();
        asm volatile ("hlt");
    }
```

### 3.2 插入显示代码

在 `for (;;)` 之前插入：

```c
    Harlin_SetMode(HARLIN_DISP_VGA_TEXT);
    Harlin_Clear();
    Harlin_Print("\n");
    Harlin_Print("Welcome to HarLin\n");
    Harlin_Print("=================\n");
```

### 3.3 重新构建并查看

```powershell
.\Build.exe
.\run-bochs.bat
```

屏幕上会显示写入的字符串。

## 四、切换到图形模式

### 4.1 支持的显示模式

```c
#define HARLIN_DISP_VGA_TEXT 0
#define HARLIN_DISP_VGA_13H  1
#define HARLIN_DISP_VESA     2
```

### 4.2 文本模式（默认）

内核启动后默认进入文本模式。使用 `Harlin_Print` 输出字符。

### 4.3 VGA 13H 图形模式

VGA 13H 是 320x200x256 色，无需 Bootloader 支持即可在内核中设置：

```c
    Harlin_SetMode(HARLIN_DISP_VGA_13H);
    Harlin_ClearScreen(0x01);
    Harlin_PutPixel(10, 10, 0x0F);
```

### 4.4 VESA 图形模式

VESA 模式需要 Bootloader 在实模式下调用 VBE BIOS。默认关闭，需要手动开启 Bootloader 中的 VBE 设置，然后在 `Harlin_Boot()` 中调用：

```c
    Harlin_SetMode(HARLIN_DISP_VESA);
```

具体启用方式请查看 `src/asm/boot/` 下的 Bootloader 源码。

### 4.5 显示 API 综合示例

```c
void Harlin_Boot(void)
{
    Harlin_SetMode(HARLIN_DISP_VGA_13H);
    Harlin_ClearScreen(0);

    for (int y = 0; y < 200; y++) {
        for (int x = 0; x < 320; x++) {
            Harlin_PutPixel(x, y, (x + y) & 0xFF);
        }
    }

    for (;;) {
        interrupts_enable();
        asm volatile ("hlt");
    }
}
```

## 五、常见坑与解决

### 5.1 构建时报 `TIMES value is negative`

**现象**：

```
boot.asm: error: TIMES value is negative
```

**原因**：Bootloader 体积超过 512 字节。

**解决**：减少 `src/asm/boot/boot.asm` 中的启动提示字符串、未使用的延时调用或调试代码。

### 5.2 Bochs 启动后灰屏/花屏

**原因**：显示模式设置与显卡状态不匹配。

**解决**：
- 确保 Bootloader 默认没有启用 VBE。
- 确保 `Harlin_Boot()` 中调用了正确的 `Harlin_SetMode`。

### 5.3 修改代码后没有生效

**原因**：运行的是旧镜像。

**解决**：每次修改后重新执行 `Build.exe`，再运行 Bochs。

### 5.4 长模式下不能调用 BIOS

**原因**：进入 64 位长模式后，BIOS 中断不可用。

**解决**：
- 显示模式：文本/VGA 13H 直接写端口；VESA 在 Bootloader 中设置。
- 磁盘：使用 ATA PIO 端口直接读写。

### 5.5 链接时报 undefined symbol

**原因**：汇编函数未被正确编译并链接。

**解决**：
- 确保 `.asm` 文件被正确包含在 `Build.exe` 的编译列表中。

### 5.6 软盘镜像启动失败

**原因**：Bochs 的 BIOS 在读取 1.44MB 软盘时存在兼容性问题。

**解决**：项目默认使用 CD-ROM（El Torito）启动。如需软盘启动，在 `bochsrc.txt` 中将 `boot:` 改为 `floppy`。

## 七、音频播放

### 7.1 初始化音频

内核支持 Sound Blaster 16 声卡（Bochs 中默认可用）。在 `Harlin_Boot()` 中调用：

```c
    if (Harlin_AudioInit() == HARLIN_OK) {
        Harlin_Print("Audio ready\n");
    }
```

### 7.2 播放 WAV 文件

```c
    extern char music_wav_data[];
    extern u32  music_wav_size;

    if (Harlin_AudioPlayWav((u8*)music_wav_data, music_wav_size) == HARLIN_OK) {
        Harlin_Print("Playing...\n");
    }
```

目前支持 16-bit PCM、单声道或立体声的 WAV 文件。文件大小不能超过 64KB。

### 7.3 停止播放

```c
    Harlin_AudioStop();
```

## 八、网络模块

### 8.1 默认启用 DHCP

网络在 `Harlin_Boot()` 中自动初始化并尝试 DHCP 获取 IP 地址。若 DHCP 超时，回退到静态 IP 配置。

### 8.2 使用网络 API

```c
    if (Harlin_InitNet() == HARLIN_OK) {
        Harlin_HttpGet("example.com", "/");
    }
```

### 8.3 TCP 多连接

内核支持同时管理多个 TCP 连接，每个连接由连接表中的独立条目跟踪。在网卡驱动中直接调用 `tcp_connect_remote`、`tcp_send`、`tcp_recv`、`tcp_close_conn` 等底层接口。

### 8.4 网络驱动架构

网络模块位于 `src/harlin/net/network.c`，包含：
- RTL8139 网卡驱动（中断驱动，接收环形缓冲区）
- ARP 协议（请求、响应、缓存表）
- IP 协议（发送、接收、校验和）
- DHCP 客户端（四步握手：Discover→Offer→Request→Ack）
- TCP 协议（连接管理、序列号跟踪、状态转换）
- DNS 客户端（域名解析）
- HTTP 客户端（HTTP/1.1 + chunked 传输编码）

## 九、多核 SMP 与同步

### 9.1 获取 CPU 信息

```c
    int count = Harlin_CpuCount();
    int cpu   = Harlin_CurrentCpu();
    Harlin_Print("CPU count: ");
    Harlin_PrintDec(count);
    Harlin_Print("\nCurrent CPU: ");
    Harlin_PrintDec(cpu);
    Harlin_Print("\n");
```

### 9.2 使用自旋锁

```c
    struct Harlin_Spinlock lk;
    Harlin_SpinlockInit(&lk);

    Harlin_SpinlockAcquire(&lk);
    /* 临界区代码 */
    Harlin_SpinlockRelease(&lk);
```

### 9.3 SMP 启动流程

1. BSP（引导处理器）初始化 GDT、IDT、PMM、VMM。
2. BSP 将 AP（应用处理器）启动代码复制到 `0x8000` 处的 trampoline 区域。
3. BSP 发送 INIT-SIPI-SIPI IPI 序列唤醒 AP。
4. 每个 AP 独立执行 trampoline，切换到保护模式→长模式，设置自己的 GDT/TSS。
5. AP 进入空闲循环等待调度。

SMP 相关源码路径：
- `src/asm/core/smp_trampoline_data.asm` — AP 启动代码
- `src/harlin/core/smp.c` — SMP 初始化与 IPI 发送
- `src/harlin/core/spinlock.c` — 自旋锁实现
- `src/harlin/core/percpu.c` — per-CPU 变量管理

## 十二、调试技巧

### 12.1 使用标记定位崩溃

在 `Harlin_Boot()` 不同阶段插入标记输出：

```c
    Harlin_Print("A\n");
    /* 初始化步骤 1 */
    Harlin_Print("B\n");
    /* 初始化步骤 2 */
    Harlin_Print("C\n");
```

观察最后一个成功输出的标记，即可定位崩溃位置。

### 12.2 使用调试端口输出

内核内部使用 0x0402 调试端口输出初始化进度。Bochs 日志 `build/bochs.log` 中可看到类似内容：

```
[KMAIN] start
[BOOT] start
[BOOT] bss filled
[BOOT] gdt done
[BOOT] idt done
...
```

这是排查早期启动崩溃的最可靠方法。

### 12.3 魔术断点

在汇编代码中插入 `xchg bx,bx`（16 位）或 `xchg ebx,ebx`（32/64 位），当 Bochs 执行到该指令时会自动暂停进入调试器。需要在 `bochsrc.txt` 中启用 `magic_break: enabled=1`。

### 12.4 单步调试

使用 `bochsdbg.exe` 启动后按 `Ctrl+C` 进入调试器：

```
b 0x7C00    ; 在引导扇区入口设断点
c           ; 运行到断点
s           ; 单步执行
r           ; 查看寄存器
xp /8bx 0x7C00  ; 查看内存
```

## 十四、ACPI 电源管理

### 14.1 自动初始化

ACPI 在 `Harlin_Boot()` 初始化后期自动检测初始化。初始化时扫描 BIOS 内存区（`0xE0000~0xFFFFF`）查找 RSDP 描述符，通过 RSDT 定位 FADT，进而获取 PM1 控制寄存器和 DSDT 中的 \_S5 软关机参数。

### 14.2 硬件关机

系统关机时自动调用 ACPI 关机（`acpi_power_off`），写入 SLP_TYPa | SLP_EN 到 PM1a_CNT 寄存器触发 S5 状态。若 ACPI 不可用，回退到 CLI + HLT。

```c
    Harlin_Shutdown();
```

### 14.3 硬件重启

系统重启优先使用 FADT 中的 RESET_REG 寄存器（I/O 端口方式），失败则回退到键盘控制器复位（`outb(0x64, 0xFE)`）。

## 十五、内存管理详解

### 15.1 物理内存管理（PMM）

PMM 使用位图追踪物理内存页。初始化时通过 INT 0x15 E820 获取内存布局，标记所有可用页。源码位于 `src/harlin/mem/pmm.c`。

关键接口：
```c
u64  Harlin_AllocPage(void);           // 分配一页 4KB
void Harlin_FreePage(u64 addr);        // 释放一页
```

### 15.2 虚拟内存管理（VMM）

VMM 管理 x86-64 四级页表。每个进程拥有独立页表，内核空间（高半区）在所有页表中共享映射。源码位于 `src/harlin/mem/vmm.c`。

关键接口：
```c
void Harlin_Map(u64 virt, u64 phys, u64 flags);     // 建立映射
void Harlin_Unmap(u64 virt);                          // 解除映射
u64  Harlin_ToPhys(u64 virt);                        // 虚拟地址转物理
```

### 15.3 内核堆分配器（kmalloc）

基于 slab/链表的内存分配器，适合小块内存分配。源码位于 `src/harlin/mem/kmalloc.c`。

```c
void* Harlin_Kmalloc(u64 size);
void  Harlin_Kfree(void* ptr);
void* Harlin_Krealloc(void* ptr, u64 size);
u64   Harlin_Ksize(void* ptr);
```

## 十六、文件系统

### 16.1 分区表

内核通过 MBR 分区表解析磁盘分区。源码位于 `src/harlin/fs/partition.c`。

```c
int Harlin_InitPart(void);
int Harlin_PartCount(void);
int Harlin_GetPart(int index, struct Harlin_Part* out);
```

### 16.2 FAT32

支持的 FAT32 操作：打开、读取、创建、写入、删除文件。目录查找支持长文件名（通过短文件名 8.3 映射）。

```c
int  Harlin_Mount(u32 partition_lba);                    // 挂载分区
int  Harlin_Open(const char* name, struct Harlin_File* out);  // 打开文件
int  Harlin_Create(const char* name, struct Harlin_File* out); // 创建文件
int  Harlin_Read(struct Harlin_File* file, void* buf, u32 len);// 读取文件
int  Harlin_Write(struct Harlin_File* file, const void* buf, u32 len);// 写入文件
void Harlin_Close(struct Harlin_File* file);             // 关闭文件
```

FAT32 实现包含自旋锁保护，支持多核安全访问。源码位于 `src/harlin/fs/fat32.c`。

### 16.3 磁盘 I/O

使用 ATA PIO 模式通过端口 0x1F0-0x3F7 操作磁盘。源码位于 `src/harlin/drv/ata.c`。

```c
int Harlin_InitDisk(void);
int Harlin_ReadSectors(u64 lba, u8 count, void* buf);
int Harlin_WriteSectors(u64 lba, u8 count, const void* buf);
```

## 十七、进程与 IPC

### 17.1 进程调度

内核支持抢占式多任务调度，每个进程拥有独立 4KB 页表。调度器使用优先级 + 时间片轮转。源码位于 `src/harlin/proc/scheduler.c`。

### 17.2 管道 IPC

进程间通信通过内核管道对象实现。源码位于 `src/harlin/proc/pipe.c`。

```c
int  Harlin_CreatePipe(struct Harlin_Pipe* pipe);
int  Harlin_ReadPipe(struct Harlin_Pipe* pipe, void* buf, u32 len);
int  Harlin_WritePipe(struct Harlin_Pipe* pipe, const void* buf, u32 len);
void Harlin_ClosePipe(struct Harlin_Pipe* pipe);
```

## 十八、扩展规范

- API 命名统一为 `Harlin_` 前缀。
- 错误码优先使用 `HARLIN_FS_ERROR`、`HARLIN_FS_EOF` 等已有定义。
- 用户态与内核态隔离，用户程序禁止直接访问 I/O 端口与物理内存。
- 硬件驱动需正确处理中断与 EOI。
- 若你对本项目存在疑惑或发现了漏洞，请报道给 HarLin。

## 十九、内存虚拟文件系统 (mem_fs)

mem_fs 是 HarLin 内核自带的纯内存目录树，不需要磁盘支持。用于存放配置、临时文件以及后续 exec 加载 elf。

### 19.1 初始化与目录

```c
Harlin_MemFsInit();
Harlin_MemFsMkdir("/etc");
Harlin_MemFsMkdir("/home");
```

### 19.2 文件读写

```c
Harlin_MemFsCreate("/etc/hostname");
Harlin_MemFsWrite("/etc/hostname", "har-lin\n", 8);

char buf[32];
int n = Harlin_MemFsRead("/etc/hostname", buf, sizeof(buf));
```

### 19.3 目录遍历

```c
struct mem_fs_dir_entry ents[16];
int n = Harlin_MemFsLs("/", ents, 16);
for (int i = 0; i < n; i++) {
    Harlin_Print(ents[i].name);
    Harlin_Print("\n");
}
```

### 19.4 权限位

```c
u16 mode = PERM_OWN_R | PERM_OWN_W | PERM_GRP_R | PERM_OTH_R;
if (Harlin_PermCheck(mode, 1000, 1000, /*uid*/0, /*gid*/0, PERM_OWN_W)) {
    // 通过
}
```

### 19.5 COW 版本

```c
char ver[8];
Harlin_CowFsInit();
Harlin_CowSnapshot("/etc/hostname", ver);  // ver = "v000"
Harlin_CowWrite("/etc/hostname", ver, "new\n", 4);
Harlin_CowReadAt("/etc/hostname", ver, 0, buf, 4);
```

### 19.6 /proc 虚拟文件

```c
char buf[64];
Harlin_ProcFsRead("/proc/uptime", buf, sizeof(buf));   // "uptime_ticks=..."
Harlin_ProcFsRead("/proc/meminfo", buf, sizeof(buf));  // "free=... used=..."
Harlin_ProcFsRead("/proc/cpuinfo", buf, sizeof(buf));  // "cpus=..."
```

## 二十、进程控制 (fork / exec / wait)

### 20.1 复制子进程

```c
int child;
Harlin_Fork(current_pid, &child);
// child 拥有与父进程相同的 rip / rsp / 句柄
```

### 20.2 加载并执行 ELF

```c
int new_pid = Harlin_ExecFromPath("/bin/init.elf");
// 或直接传缓冲区
Harlin_ExecFromBuffer(elf_data, elf_size);
```

### 20.3 等待子进程退出

```c
Harlin_NotifyExit(child, 0);  // 由子进程在退出时调用
int code;
if (Harlin_Wait(current_pid, child, &code) == 0) {
    // 拿到退出码
}

// 任意子进程退出
int any_pid, any_code;
Harlin_WaitAny(current_pid, &any_pid, &any_code);
```

### 20.4 事件通知

```c
Harlin_NotifySend(target_pid, 1);
int ev;
Harlin_NotifyWait(current_pid, &ev);
```
