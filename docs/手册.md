# HarLin Kernel 二次开发手册

本手册面向基于 HarLin Kernel 进行扩展开发的第三方开发者。

## 开发环境

- Windows 10/11
- NASM 2.14+
- MinGW-w64 x86_64 GCC
- GNU Binutils LD / objcopy
- QEMU（可选，用于运行调试）

工具链路径已在 build.bat 中指定为 `D:\Code\Local_tool_library\`，可按实际环境修改。

## 项目结构

```
HarLIn_Boot/
├── build/          构建产物（HarLin.img、kernel.bin 等）
├── docs/           文档
├── src/
│   ├── ASM/        启动与底层汇编
│   ├── Sys_C/      C 语言内核模块
│   └── head/       头文件与 API 声明
├── build.bat       编译脚本
└── run.bat         QEMU 运行脚本
```

## 构建内核

```bat
build.bat
```

构建成功会生成 `build/HarLin.img`。运行：

```bat
run.bat
```

## 添加内核模块

### 1. 新增 C 驱动或功能

在 `src/Sys_C/` 新建源文件，例如 `mydriver.c`，在 `src/head/` 创建对应头文件。在 `harlin_API.c` 的 `Harlin_Boot()` 中调用初始化函数，并通过 `harlin_API.h` 暴露接口。

### 2. 新增系统调用

编辑 `src/head/harlin_API.h` 中的系统调用号枚举：

```c
enum {
    HARLIN_SYS_EXIT = 0,
    HARLIN_SYS_PRINT,
    HARLIN_SYS_MYCALL,
};
```

在 `src/Sys_C/syscall.c` 中实现处理函数并填入 `syscall_table`：

```c
static unsigned long sys_mycall(struct syscall_regs* r)
{
    return r->rdi + r->rsi;
}

static syscall_t syscall_table[] = {
    [HARLIN_SYS_EXIT] = sys_exit,
    [HARLIN_SYS_PRINT] = sys_putchar,
    [HARLIN_SYS_MYCALL] = sys_mycall,
};
```

用户态程序通过 `int 0x80` 调用，调用号置于 `rax`，参数置于 `rdi`、`rsi`、`rdx`。

### 3. 添加中断处理

在 `src/Sys_C/interrupt.c` 中调用 `irq_register(irq_number, handler)` 注册中断处理函数。处理函数执行完毕后，若使用 PIC，需调用 `pic_eoi(irq_number)`。

## 开发用户态 .cx 程序

HarLin Kernel 使用自定义 .cx 可执行格式，不兼容 ELF。程序入口必须为 64 位长模式代码。

最小可执行文件结构：

| 字段 | 说明 |
|------|------|
| magic | 8 字节 `"HARLINCX"` |
| version | 必须为 1 |
| entry_offset | 代码段内入口偏移 |
| code_offset / code_size | 代码在文件中的位置与大小 |
| data_offset / data_size | 只读数据位置与大小（可选） |
| bss_size | 未初始化数据大小（可选） |
| reloc_offset / reloc_count | 重定位表（可选） |
| stack_size | 栈大小（可选） |

程序加载基址为 `0x400000`，栈顶为 `0x800000`。用户态代码运行在 ring3，必须通过系统调用访问内核功能。

用户态最小示例：

```asm
section .text
global _start
_start:
    mov rax, 1          ; HARLIN_SYS_PRINT
    mov rdi, 'A'
    int 0x80

    mov rax, 0          ; HARLIN_SYS_EXIT
    int 0x80
```

将上述汇编编译为 .cx 格式并放置于磁盘镜像根目录，文件名为 `init.cx`。内核启动时会自动加载并运行。

## 常用 API

所有 API 声明位于 `src/head/harlin_API.h`，主要分类如下：

- 控制台：`Harlin_ConPrint`、`Harlin_ConClear`
- 键盘：`Harlin_KeyReady`、`Harlin_KeyGet`
- 内存：`Harlin_PmmAlloc`、`Harlin_PmmFree`、`Harlin_VmmMap`
- 磁盘：`Harlin_DiskReadSector`、`Harlin_DiskWriteSector`
- 文件系统：`Harlin_FsMount`、`Harlin_FsOpen`、`Harlin_FsRead`
- 网络：`Harlin_NetInit`、`Harlin_HttpGet`

## 调试建议

1. 使用 `screen_put_char` 或 `Harlin_ConPrint` 输出字符到 VGA 文本缓冲区。
2. QEMU 运行时查看串口或调试输出，可在 run.bat 中调整 `-d` 参数。
3. 新增代码后先单独编译确认无警告，再运行完整构建。

## 扩展规范

- API 命名统一为 `Harlin_` 前缀。
- 错误码优先使用 `HARLIN_FS_ERROR`、`HARLIN_FS_EOF` 等已有定义。
- 用户态与内核态隔离，用户程序禁止直接访问 I/O 端口与物理内存。
- 新增硬件驱动需正确处理中断与 EOI。
