# HarLin Kernel 手册

本手册面向第一次接触 HarLin Kernel 的开发者。按照章节顺序操作，即可从零开始构建、修改并运行内核。

***

## 一、环境准备

### 1.1 需要的工具

| 工具            | 用途       | 推荐版本              |
| ------------- | -------- | ----------------- |
| NASM          | 编译汇编源文件  | 2.14+             |
| MinGW-w64 GCC | 编译 C 源文件 | x86_64-posix-seh |
| GNU Binutils  | 链接与生成二进制 | ld、objcopy        |
| QEMU（可选）      | 运行与调试    | 7.0+              |

将上述工具的安装目录加入系统 PATH，或在 `build.bat` 与 `run.bat` 中直接修改 PATH 变量。

### 1.2 目录结构

```
Kernel/
├── build/              构建产物
├── docs/               文档
├── src/                源码
│   ├── asm/            启动与底层汇编
│   ├── harlin/         C 语言内核模块
│   └── head/           头文件与 API 声明
├── hcc/                HCC 编译器源码
├── bin/                HCC 编译器可执行文件
├── build.bat           一键编译脚本
└── run.bat             QEMU 运行脚本
```

## 二、第一次构建与运行

### 2.1 构建

打开 PowerShell，进入项目目录：

```powershell
cd Kernel
.\build.bat
```

如果工具链配置正确，构建成功会显示：

```
Build successful
```

产物在 `build/HarLin.img`。

如果构建失败，请检查：

- NASM、GCC、ld 是否在 PATH 中。
- `build.bat` 中的工具路径是否与实际安装位置一致。

### 2.2 运行

```powershell
.\run.bat
```

内核默认纯黑屏启动，不显示任何输出。QEMU 窗口为纯黑屏且无崩溃，即表示内核已成功启动。

如需添加启动画面，请参考第 3 章。如需切换到图形模式，请参考第 4 章。

如果看到灰色/花屏，说明显示模式设置异常，请跳到第 5 章排查。

### 2.3 重新构建的习惯

每次修改源码后，都必须重新执行 `.\build.bat`，再运行 `.\run.bat`。如果运行的是旧镜像，修改不会生效。

## 三、如何添加启动画面

内核默认不显示任何文字。添加启动画面只需要在启动流程中调用显示 API。

### 3.1 打开启动入口

内核启动入口位于 `src/harlin/harlin_API.c`，找到文件末尾附近的闲置循环：

```c
    for (;;) {
        interrupts_enable();
        asm volatile ("hlt");
    }
```

### 3.2 插入显示代码

在 `for (;;)` 之前插入：

```c
    Harlin_SetMode(HARLIN_DISP_VGA_TEXT);
    Harlin_Clear();
    Harlin_Print("\n");
    Harlin_Print("Welcome to HarLin\n");
    Harlin_Print("=================\n");
```

### 3.3 重新构建并查看

```powershell
.\build.bat
.\run.bat
```

运行后，屏幕上会显示你写入的字符串。

## 四、切换到图形模式

### 4.1 支持的显示模式

```c
#define HARLIN_DISP_VGA_TEXT 0
#define HARLIN_DISP_VGA_13H  1
#define HARLIN_DISP_VESA     2
```

### 4.2 文本模式

内核启动后默认进入文本模式。在文本模式下，使用 `Harlin_Print` 输出字符。

### 4.3 VGA 13H 图形模式

VGA 13H 是 320x200x256 色，无需 Bootloader 支持即可在内核中设置。

在启动入口中调用：

```c
    Harlin_SetMode(HARLIN_DISP_VGA_13H);
    Harlin_ClearScreen(0x01);
    Harlin_PutPixel(10, 10, 0x0F);
```

### 4.4 VESA 图形模式

VESA 模式需要 Bootloader 在实模式下调用 VBE BIOS。默认关闭，需要手动开启 Bootloader 中的 VBE 设置，然后在 `Harlin_Boot()` 中调用：

```c
    Harlin_SetMode(HARLIN_DISP_VESA);
```

具体启用方式请查看 Bootloader 源码中的 VBE 相关函数。

### 4.5 显示 API 示例

```c
void Harlin_Boot(void)
{
    /* 其他初始化 */

    Harlin_SetMode(HARLIN_DISP_VGA_13H);
    Harlin_ClearScreen(0);

    for (int y = 0; y < 200; y++) {
        for (int x = 0; x < 320; x++) {
            Harlin_PutPixel(x, y, (x + y) & 0xFF);
        }
    }

    for (;;) {
        interrupts_enable();
        asm volatile ("hlt");
    }
}
```

## 五、常见坑与解决

### 5.1 构建时报 `TIMES value is negative`

**现象**：

```
boot.asm: error: TIMES value is negative
```

**原因**：Bootloader 体积超过 512 字节。

**解决**：减少 `src/asm/boot.asm` 中的启动提示字符串、未使用的延时调用或调试代码。

### 5.2 QEMU 启动后灰屏/花屏

**原因**：显示模式设置与显卡状态不匹配。

**解决**：

- 确保 Bootloader 默认没有启用 VBE。
- 确保 `Harlin_Boot()` 中调用了正确的 `Harlin_SetMode`。

### 5.3 修改代码后没有生效

**原因**：运行的是旧镜像。

**解决**：每次修改后重新执行 `.\build.bat`，再运行 `.\run.bat`。

### 5.4 长模式下不能调用 BIOS

**原因**：进入 64 位长模式后，BIOS 中断不可用。

**解决**：

- 显示模式：文本/VGA 13H 直接写端口；VESA 在 Bootloader 中设置。
- 磁盘：使用 ATA PIO 端口直接读写。

## 七、网络模块

### 7.1 默认不启用

网络默认不在 `Harlin_Boot()` 中初始化，以防止某些环境下启动卡住。

### 7.2 启用网络

在 `Harlin_Boot()` 中，找到 `keyboard_init()` 和 `interrupts_enable();` 之间，插入：

```c
    keyboard_init();
    network_init();
    interrupts_enable();
```

### 7.3 使用网络 API

```c
    if (Harlin_InitNet() == HARLIN_OK) {
        Harlin_HttpGet("example.com", "/");
    }
```

网络配置（IP、网关、DNS）在网卡驱动源码中设置，QEMU 默认使用 RTL8139 网卡。

## 八、多核 SMP 与同步

### 8.1 获取 CPU 信息

```c
    int count = Harlin_CpuCount();
    int cpu   = Harlin_CurrentCpu();
    Harlin_Print("CPU count: ");
    Harlin_PrintDec(count);
    Harlin_Print("\nCurrent CPU: ");
    Harlin_PrintDec(cpu);
    Harlin_Print("\n");
```

### 8.2 使用自旋锁

```c
    struct Harlin_Spinlock lk;
    Harlin_SpinlockInit(&lk);

    Harlin_SpinlockAcquire(&lk);
    /* 临界区代码 */
    Harlin_SpinlockRelease(&lk);
```

## 九、动态内存与 RTC

### 9.1 使用 kmalloc

```c
    char* buf = (char*)Harlin_Kmalloc(256);
    if (buf) {
        Harlin_Copy(buf, "Hello", 6);
        Harlin_Print(buf);
        Harlin_Kfree(buf);
    }
```

### 9.2 读取 RTC 时间

```c
    struct Harlin_RtcTime t;
    Harlin_RtcRead(&t);

    Harlin_PrintDec(t.year);
    Harlin_Print("-");
    Harlin_PrintDec(t.month);
    Harlin_Print("-");
    Harlin_PrintDec(t.day);
    Harlin_Print(" ");
    Harlin_PrintDec(t.hour);
    Harlin_Print(":");
    Harlin_PrintDec(t.minute);
    Harlin_Print(":");
    Harlin_PrintDec(t.second);
    Harlin_Print("\n");
```

## 十、键盘与系统调用

## 十一、调试技巧

### 11.1 使用标记定位崩溃

在 `Harlin_Boot()` 不同阶段插入标记输出：

```c
    Harlin_Print("A\n");
    /* 初始化步骤 1 */
    Harlin_Print("B\n");
    /* 初始化步骤 2 */
    Harlin_Print("C\n");
```

观察最后一个成功输出的标记，即可定位崩溃位置。

### 11.2 使用串口输出

如果屏幕不可用，可在代码中加入串口输出（需要串口驱动支持）。

### 11.3 QEMU 调试

`run.bat` 默认以图形窗口启动。如需更多调试信息，可在 `run.bat` 中添加 QEMU 调试参数。

***

## 十二、扩展规范

- API 命名统一为 `Harlin_` 前缀。
- 错误码优先使用 `HARLIN_FS_ERROR`、`HARLIN_FS_EOF` 等已有定义。
- 用户态与内核态隔离，用户程序禁止直接访问 I/O 端口与物理内存。
- 硬件驱动需正确处理中断与 EOI。
- 禁止在代码、注释、文件名中写入 HarLin228 Studio 以外的第三方品牌与名称。
- 若你对本项目存在疑惑或发现了漏洞，请报道给 HarLin。
