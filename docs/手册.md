# HarLin Kernel 手册

本手册面向第一次接触 HarLin Kernel 的开发者。按照章节顺序操作，即可从零开始构建、修改并运行内核。

***

## 一、环境准备

### 1.1 需要的工具

| 工具            | 用途       | 推荐版本              |
| ------------- | -------- | ----------------- |
| NASM          | 编译汇编源文件  | 2.14+             |
| MinGW-w64 GCC | 编译 C 源文件 | x86\_64-posix-seh |
| GNU Binutils  | 链接与生成二进制 | ld、objcopy        |
| QEMU（可选）      | 运行与调试    | 7.0+              |

工具链已预设路径为 `D:\Code\Local_tool_library\`。若你的工具放在其他位置，请修改 `build.bat` 与 `run.bat` 中的 PATH。

### 1.2 目录结构

```
Kernel/
├── build/              构建产物
│   ├── boot.bin
│   ├── kernel.bin
│   └── HarLin.img      最终软盘镜像
├── docs/               文档（你正在看的就是这里）
├── src/
│   ├── asm/            启动与底层汇编
│   ├── harlin/         C 语言内核模块
│   └── head/           头文件与 API 声明
├── hcc/                HCC 编译器源码
├── bin/                HCC 编译器可执行文件
├── build.bat           一键编译脚本
└── run.bat             QEMU 运行脚本
```

## 二、第一次构建与运行

### 2.1 构建

打开 PowerShell，进入项目目录：

```powershell
cd D:\Code\Code\Kernel
.\build.bat
```

构建成功会显示：

```
Build successful
```

产物在 `build/HarLin.img`。

### 2.2 运行

```powershell
.\run.bat
```

内核默认纯黑屏启动，不显示任何输出。QEMU 窗口为纯黑屏且无崩溃，即表示内核已成功启动。

如需添加启动画面，请参考第 3 章。如需切换到图形模式，请参考第 4 章。

如果看到灰色/花屏，说明显示模式设置异常，请跳到第 5 章排查。

***

## 三、如何添加启动画面

内核默认不显示任何文字。如需添加启动画面，需要在 `src/harlin/harlin_API.c` 的 `Harlin_Boot()` 函数中添加显示代码。

### 3.1 定位代码

打开 `src/harlin/harlin_API.c`，翻到文件末尾附近，找到闲置循环：

```c
    for (;;) {
        Harlin_IntOn();
        asm volatile ("hlt");
    }
```

### 3.2 添加启动画面

在 `for (;;)` 之前插入显示代码：

```c
    Harlin_SetMode(HARLIN_DISP_VGA_TEXT);
    Harlin_Clear();
    Harlin_Print("\n");
    Harlin_Print("My OS\n");
    Harlin_Print("------------------------\n");
```

`Harlin_Print` 中的字符串可以改成任何你想要的文字。

### 3.3 重新构建

保存后执行 `.\build.bat`，再用 `.\run.bat` 查看效果。

## 四、切换到图形模式

内核支持三种显示模式：

```c
#define HARLIN_DISP_VGA_TEXT 0
#define HARLIN_DISP_VGA_13H  1
#define HARLIN_DISP_VESA     2
```

### 4.1 默认为什么是文本模式

Bootloader 第一阶段位于 512 字节的主引导扇区，空间非常紧张。VBE 初始化代码虽然存在，但默认不调用，以保证内核启动后一定处于稳定的 VGA 文本模式。

### 4.2 切换到 VGA 13H 模式

VGA 13H 是 320x200x256 色，可直接在内核中通过写 VGA 寄存器设置，无需 Bootloader 支持。

打开 `src/harlin/harlin_API.c`，在 `for (;;)` 之前添加或修改显示模式设置：

```c
    Harlin_SetMode(HARLIN_DISP_VGA_13H);
```

重新构建运行，屏幕会进入 320x200 图形模式。

### 4.3 切换到 VESA 800x600 模式

VESA 需要 Bootloader 在实模式下调用 VBE BIOS。默认已关闭，需要手动开启。

#### 步骤 1：启用 Bootloader VBE

打开 `src/asm/boot.asm`，翻到第 60 行左右：

```asm
    mov si, msg_vbe
    call print_string
    mov si, msg_ok
    call print_string
```

在第 61 行之后插入：

```asm
    call setup_vesa
```

变成：

```asm
    mov si, msg_vbe
    call print_string
    call setup_vesa
    mov si, msg_ok
    call print_string
```

#### 步骤 2：修改内核启动代码

打开 `src/harlin/harlin_API.c`，在 `for (;;)` 之前添加或修改显示模式设置：

```c
     Harlin_SetMode(HARLIN_DISP_VESA);
```

#### 步骤 3：重新构建运行

执行 `.\build.bat`，然后 `.\run.bat`。

#### 注意

VESA 模式号使用标准 VBE 2.0 的 `0x115`（800x600x24bpp），在 `src/asm/boot.asm` 第 147 行定义：

```asm
    mov bx, 0x4115
```

`0x4115` = `0x115 | 0x4000`，其中 `0x4000` 表示使用线性帧缓冲（LFB）。

## 五、常见坑与解决

### 5.1 构建时报 `TIMES value is negative`

**现象**：

```
boot.asm:203: error: TIMES value -8 is negative
```

**原因**：Bootloader 体积超过 512 字节。

**解决**：减少 `src/asm/boot.asm` 中的字符串或指令。常见可删内容：

- 较长的启动提示字符串
- 未使用的 `delay_1s` 调用
- 多余的空行不影响大小，但调试代码会影响

### 5.2 QEMU 启动后灰屏/花屏

**原因 1**：Bootloader 默认设置了 VBE 图形模式，但内核没有正确切回文本模式。

**解决**：确保 `src/asm/boot.asm` 中没有调用 `setup_vesa`（默认已移除），且内核 `Harlin_Boot()` 中未设置错误的显示模式。

**原因 2**：VGA 文本模式寄存器初始化异常。

**解决**：检查 `src/harlin/display.c` 中的 `vga_set_mode_text()` 是否被 `display_set_mode()` 调用（第 222 行附近）。

### 5.3 内核启动后显示异常

**现象**：QEMU 窗口出现乱码、闪烁或异常彩色。

**原因**：内核显示模式与显卡状态不匹配。

**解决**：确保未在 `Harlin_Boot()` 中错误调用 `Harlin_SetMode`，或参考第 4 章正确设置显示模式。

### 5.4 修改代码后没有生效

**原因**：可能运行的是旧镜像。

**解决**：每次修改后必须重新执行 `.\build.bat`，再运行 `.\run.bat`。

### 5.5 添加系统调用后编译失败

**原因**：系统调用号、处理函数、syscall\_table 三者未对齐。

**正确流程**：

1. 在 `src/head/harlin_API.h` 中新增调用号（参考第 42 行开始的 `HARLIN_SYS_*` 枚举）。
2. 在 `src/harlin/syscall.c` 中实现处理函数。
3. 在 `syscall_table[]` 的对应下标位置填入函数指针。

例如新增 `HARLIN_SYS_BEEP`：

```c
// src/head/harlin_API.h，第 42 行附近
#define HARLIN_SYS_PIPE_READY   16
#define HARLIN_SYS_BEEP         17
```

```c
// src/harlin/syscall.c
static unsigned long sys_beep(struct syscall_regs* r)
{
    speaker_beep((u32)r->rdi);
    return 0;
}

static syscall_t syscall_table[] = {
    ...
    [HARLIN_SYS_PIPE_READY] = sys_pipe_ready,
    [HARLIN_SYS_BEEP]       = sys_beep,
};
```

### 5.6 长模式下不能调用 BIOS

**原因**：进入 64 位长模式后，`int 0x10` 等 BIOS 中断不可用。

**影响**：VBE 设置、磁盘读写等必须在 Bootloader 实模式阶段完成，或通过直接操作硬件端口完成。

**解决**：

- 显示模式：文本/VGA 13H 直接写端口；VESA 在 Bootloader 中设置。
- 磁盘：使用 ATA PIO 端口直接读写，不依赖 BIOS。

## 六、添加一个新功能的标准流程

以"开机时蜂鸣一声"为例：

### 6.1 确定入口

功能应在 `src/harlin/harlin_API.c` 的 `Harlin_Boot()` 中初始化完成后调用。

### 6.2 找到现有实现

`src/asm/speaker.asm` 已有 `beep` 函数，在 Bootloader 中使用。内核中可调用 `speaker_beep`。

### 6.3 添加调用

在 `src/harlin/harlin_API.c` 的 `for (;;)` 之前插入：

```c
    speaker_beep(500);
```

如果缺少声明，在 `src/head/harlin_API.h` 中添加：

```c
void speaker_beep(u32 freq);
```

### 6.4 构建运行

```powershell
.\build.bat
.\run.bat
```

## 七、网络模块

### 7.1 默认不启用

为保证内核地基稳定启动，默认不在 `Harlin_Boot()` 中初始化网络。`src/harlin/harlin_API.c` 中原本存在的一行：

```c
network_init();
```

已被移除。

### 7.2 为什么默认不启用

网络初始化需要向默认网关 `10.0.2.2` 发送 ARP 请求并等待响应。在某些虚拟环境或真实硬件中，这个等待过程可能极长甚至无限，导致内核卡死在启动阶段。

### 7.3 如何启用网络

如果你确实需要网络功能，按以下步骤操作：

#### 步骤 1：打开启动入口

打开 `src/harlin/harlin_API.c`，翻到大约第 321 行，找到：

```c
    keyboard_init();
    interrupts_enable();
```

#### 步骤 2：插入网络初始化

在 `keyboard_init();` 和 `interrupts_enable();` 之间插入：

```c
    keyboard_init();
    network_init();
    interrupts_enable();
```

#### 步骤 3：使用网络 API

网络初始化成功后，即可调用：

```c
Harlin_HttpGet("example.com", "/");
```

### 7.4 网络环境要求

- 当前驱动基于 RTL8139 网卡
- QEMU 默认网卡即为 RTL8139，可直接使用
- 默认 IP：`10.0.2.15`
- 默认网关：`10.0.2.2`
- 默认 DNS：`10.0.2.3`

### 7.5 网络卡住的排查

如果启用网络后启动卡住：

1. 确认 QEMU 网络参数包含 `-netdev user,id=net0 -device rtl8139,netdev=net0`
2. 检查 `run.bat` 中是否正确设置了网络
3. 如果仍然卡住，移除 `network_init()` 恢复默认启动

## 八、调试技巧

### 8.1 使用串口输出

如果屏幕不可用，可在代码中加入串口输出（需要 `serial.c` 支持）。

### 8.2 使用 QEMU 调试

`run.bat` 默认以图形窗口启动。如需查看寄存器输出，可添加：

```bat
qemu-system-x86_64 -d int,cpu_reset -no-reboot -no-shutdown ...
```

如果有奇怪的文字或难以解决的问题，请不要惊慌，你可以选择发给 Kimi 大模型询问问题或上网查询。

### 8.3 分步定位问题

1. 确认 `build/boot.bin` 和 `build/kernel.bin` 生成时间是最新的。
2. 在 `Harlin_Boot()` 不同阶段插入 `Harlin_Print("A\n");` 等标记。
3. 观察最后一个成功输出的标记，即可定位崩溃位置。

## 九、扩展规范

- API 命名统一为 `Harlin_` 前缀。
- 错误码优先使用 `HARLIN_FS_ERROR`、`HARLIN_FS_EOF` 等已有定义。
- 用户态与内核态隔离，用户程序禁止直接访问 I/O 端口与物理内存。
- 新增硬件驱动需正确处理中断与 EOI。
- 禁止在代码、注释、文件名中写入 HarLin228 Studio 以外的第三方品牌与名称。
- 若你对本项目存在疑惑或发现了漏洞，请报道给HarLin，虽然你不会得到报酬，但HarLin感谢你的做法 并将你写进贡献清单
