# HarLin Kernel 实践教程

本文档提供从入门到进阶的实践教程，帮助开发者快速上手 HarLin Kernel 的构建、调试和功能扩展。

***

## 教程一：环境搭建与首次构建

### 1.1 准备工具

在开始之前，确保已安装以下工具并加入 PATH：

| 工具 | 用途 | 下载 |
|------|------|------|
| NASM 2.14+ | 汇编编译器 | https://www.nasm.us/ |
| LLVM 18+ (clang + ld.lld) | C 编译器 + 链接器 | https://github.com/llvm/llvm-project |
| GNU Binutils (objcopy, strip) | 二进制处理 | LLVM 附带的 llvm-objcopy |
| Bochs 2.7+ | 模拟运行和调试 | https://sourceforge.net/projects/bochs/files/bochs/ |

### 1.2 首次构建

打开 PowerShell，进入内核项目目录：

```
cd D:\Code\Code\Kernel
.\build.bat
```

构建过程将依次：
1. 编译 `boot.asm` → `build/Binary/boot.bin`
2. 编译 `stage2.asm` → `build/Binary/stage2.bin`
3. 编译所有汇编模块 → `build/asm_*.o`
4. 编译所有 C 模块 → `build/*.o`
5. 链接生成 `build/Binary/kernel.bin`
6. 拼接生成 `build/HarLin.iso`

每个步骤成功会显示 `[Success] [N/Total]`，失败会显示 `[Failure]` 并输出错误日志。

### 1.3 在 Bochs 中运行

确保 `D:\Bochs-3.0\` 已安装 Bochs，并包含 `BIOS-bochs-latest` 和 `VGABIOS-lgpl-latest.bin`。然后：

```
bochsdbg.exe -q -f bochsrc.txt
```

- 启动后按 `c` 键继续执行
- 内核启动后将显示 Shell 界面
- 可通过 0xE9 端口查看调试输出（Bochs 控制台）

## 教程二：使用 Bochs 调试内核

### 2.1 设置断点

在 Bochs 调试器中：

```
# 在物理地址设断点
pb 0x7C00            # boot.asm 入口
pb 0x7E00            # stage2 入口
pb 0x10000           # kernel_main 入口

# 在虚拟地址设断点
vb 0xFFFF800000000000 # 内核堆起始地址

# 魔术断点（在源码中插入 xchg bx,bx）
# 执行到 xchg bx,bx 时会自动暂停
```

### 2.2 单步执行常用命令

```
s         — 单步执行 (step)
n         — 单步跳过函数调用 (next)
c         — 继续执行 (continue)
 Ctrl+C   — 中断执行

reg       — 查看 CPU 寄存器
r          — 同上
sreg      — 查看段寄存器
creg      — 查看控制寄存器
dreg      — 查看调试寄存器
```

### 2.3 查看内存

```
xp /NxF 物理地址    — 查看物理内存
  xp /10x 0x7C00   — 查看 boot.asm 前 10 个 dword
x /NxF 线性地址     — 查看线性地址
  x /40x 0xB8000   — 查看 VGA 显存
```

### 2.4 查看页表

```
info tab        — 打印页表结构
page 线性地址    — 查询线性地址的页表映射
  page 0x400000 — 查看用户程序地址的页表
```

### 2.5 使用调试端口

内核通过 `0x0402` 端口输出调试信息。在 Bochs 中，该端口映射到控制台输出。可以通过 `outb 0xE9, al` 直接输出单字符。

在内核启动过程中，观察 0xE9 端口的输出序列可以快速定位初始化失败的位置：

```
输出 B      → boot.asm 正在执行
输出 R      → A20 已启用
输出 L      → 正在加载 stage2
输出 J      → 即将跳转到 stage2
输出 XYZ    → stage2 已启动
输出 S      → 正在加载内核
输出 K      → 内核加载完成
输出 P      → 已进入保护模式
```

如果输出停在某个字符之前，说明对应步骤出错。

### 2.6 常见调试场景

**场景：内核启动后无显示**

1. 检查 `0xE9` 端口输出，确定卡在哪个阶段
2. 如果是 boot.asm 阶段，检查磁盘镜像是否正确
3. 如果是 stage2 阶段，检查 GDT 和页表设置
4. 如果是内核阶段，检查链接脚本和入口符号

**场景：Page Fault 异常**

1. 在 IDT 入口 14（#PF）处设断点
2. 查看 CR2 寄存器获取出错的线性地址
3. 检查页表映射是否正确

**场景：进程切换异常**

1. 在 `timer_handler` 处设断点
2. 单步观察上下文保存和恢复过程
3. 检查 TSS 中的 RSP0 是否正确定义

## 教程三：添加新的系统调用

### 3.1 在 syscall.c 中注册

打开 `src/harlin/syscall/syscall.c`：

```c
// 1. 定义新系统调用的处理函数
static unsigned long sys_mycall(struct syscall_regs* r)
{
    // r->rdi = 第一个参数
    // r->rsi = 第二个参数
    // 返回值通过 rax 传递
    
    int arg = (int)r->rdi;
    // ... 实现逻辑
    
    return result;
}

// 2. 在 syscall_table 数组中注册
static syscall_t syscall_table[] = {
    [0] = sys_exit,
    [1] = sys_write,
    // ... 现有条目 ...
    [N] = sys_mycall,   // 新增
};
```

### 3.2 在 API 头文件中声明

打开 `src/head/syscall/harlin_API.h`：

```c
// 添加用户态 API 声明
int Harlin_MyCall(int arg);
```

### 3.3 在 harlin_API.c 中实现包装函数

打开 `src/harlin/syscall/harlin_API.c`：

```c
int Harlin_MyCall(int arg)
{
    int result;
    // 通过 int 0x80 触发系统调用
    asm volatile (
        "int $0x80"
        : "=a"(result)
        : "a"((unsigned long)N), "D"((unsigned long)arg)
        : "memory"
    );
    return result;
}
```

### 3.4 重新构建

```
.\build.bat
```

## 教程四：编写 CHC 用户程序

### 4.1 程序结构

CHC 格式用户程序运行在 ring 3 用户态，通过 `int 0x80` 调用内核服务。

```c
// hello.chc
void _start(void)
{
    // 系统调用 1 = write
    // 参数 rdi = 文件描述符 (1 = stdout)
    // 参数 rsi = 缓冲区地址
    // 参数 rdx = 长度
    char msg[] = "Hello from ring 3!\n";
    
    asm volatile (
        "mov $1, %%rax\n"    // syscall number: write
        "mov $1, %%rdi\n"    // fd: stdout
        "mov %0, %%rsi\n"    // buf
        "mov %1, %%rdx\n"    // count
        "int $0x80\n"
        :
        : "r"(msg), "r"(sizeof(msg) - 1)
        : "rax", "rdi", "rsi", "rdx", "memory"
    );
    
    // 系统调用 0 = exit
    asm volatile (
        "xor %%rax, %%rax\n"
        "int $0x80\n"
        :
        :
        : "rax"
    );
}
```

### 4.2 编译用户程序

使用 i686 或 x86_64 裸机工具链编译（不要链接标准库）：

```
clang -ffreestanding -nostdlib -static -target x86_64-unknown-none-elf -o hello hello.chc
llvm-objcopy -O binary hello hello.bin
```

### 4.3 加载到内核中运行

在内核中通过以下方式加载用户程序：

```c
// 在 shell 或调度器中调用
int pid = create_process(program_addr, program_size);
if (pid >= 0) {
    scheduler_start(pid);  // 开始调度执行
}
```

### 4.4 可用的系统调用

| 编号 | 函数 | 功能 |
|------|------|------|
| 0 | exit | 退出进程 |
| 1 | write | 写入文件/设备 |
| 2 | read | 读取文件/设备 |
| 3 | open | 打开文件 |
| 4 | close | 关闭文件 |
| 5 | exec | 执行程序 |

## 教程五：内核网络编程

### 5.1 初始化网络

```c
// 在内核模块中使用网络功能
if (Harlin_InitNet() == 0) {
    // 网络初始化成功
    // 默认使用 DHCP 获取 IP (10.0.2.0/24 子网)
}
```

### 5.2 DNS 解析

```c
u8 ip[4];
if (Harlin_Resolve("example.com", ip) == 0) {
    // ip[0]~ip[3] 包含解析后的 IPv4 地址
    Harlin_Print("Resolved IP: ");
    Harlin_PrintDec(ip[0]); Harlin_PutChar('.');
    Harlin_PrintDec(ip[1]); Harlin_PutChar('.');
    Harlin_PrintDec(ip[2]); Harlin_PutChar('.');
    Harlin_PrintDec(ip[3]); Harlin_PutChar('\n');
}
```

### 5.3 HTTP GET 请求

```c
if (Harlin_HttpGet("example.com", "/index.html") == 0) {
    // HTTP 请求成功，响应数据在内部缓冲区
}
```

### 5.4 协议栈架构

```
应用层     ─  HTTP (GET)
传输层     ─  TCP (seq/ack, 状态机)
网络层     ─  IP (v4, 路由)
链路层     ─  ARP (MAC 解析)
物理层     ─  RTL8139 (PCI, IO 端口)
```

### 5.5 Bochs 网络配置

Bochs 中使用 RTL8139 网卡实现虚拟网络：

```
# 在 bochsrc.txt 中添加
pci: enabled=1, chipset=i440fx
ata0: enabled=1, ioaddr1=0x1f0, ioaddr2=0x3f0, irq=14
```

Bochs 默认在 `10.0.2.0/24` 子网中提供 DHCP 和网关服务。

## 教程六：多核编程

### 6.1 检查多核环境

```c
// 获取 CPU 数量
int ncpus = Harlin_CpuCount();
Harlin_Print("Number of CPUs: ");
Harlin_PrintDec(ncpus);
Harlin_PutChar('\n');

// 获取当前 CPU ID
int cpu_id = Harlin_CurrentCpu();
```

### 6.2 使用自旋锁

```c
// 定义全局锁
static struct Harlin_Spinlock my_lock;

// 初始化（在初始化阶段调用一次）
Harlin_SpinlockInit(&my_lock);

// 在关键区段使用
Harlin_SpinlockAcquire(&my_lock);
// ... 受保护的共享数据访问 ...
Harlin_SpinlockRelease(&my_lock);
```

### 6.3 跨 CPU 核间中断 (IPI)

```c
// 向指定 CPU 发送 IPI
// vector = 要触发的中断向量号
Harlin_SendIpi(cpu_id, vector_number);
```

### 6.4 SMP 启动原理

1. BSP (引导处理器) 执行标准初始化流程
2. 在 `smp_init()` 中通过 INIT-SIPI-SIPI IPI 序列唤醒 AP (应用处理器)
3. AP 执行 `smp_trampoline.asm` 进入长模式
4. AP 完成各自初始化后进入空闲循环等待调度

## 教程七：内存管理实践

### 7.1 查看物理内存信息

```c
// 分配一个 4KB 物理页
u64 page = Harlin_AllocPage();
if (page != 0) {
    // 使用 page 地址
    // ...
    // 释放
    Harlin_FreePage(page);
}
```

### 7.2 使用内核堆分配器

```c
// 分配堆内存（位于 KERNEL_HEAP_START 区域）
void* buf = Harlin_Kmalloc(1024);
if (buf) {
    Harlin_Fill(buf, 0, 1024);  // 清零
    // ... 使用 ...
    Harlin_Kfree(buf);
}

// 调整大小
void* new_buf = Harlin_Krealloc(buf, 2048);
if (new_buf) {
    buf = new_buf;
}

// 查询分配大小
u64 size = Harlin_Ksize(buf);
```

### 7.3 虚拟内存映射

```c
// 映射一个物理页到虚拟地址空间
u64 phys = Harlin_AllocPage();
Harlin_Map(virt_addr, phys, VMM_PRESENT | VMM_WRITABLE);

// 查询虚拟地址对应的物理地址
u64 real_phys = Harlin_ToPhys(virt_addr);

// 解除映射
Harlin_Unmap(virt_addr);
```

### 7.4 内存分配层级

```
Harlin_Kmalloc()       — 堆分配器 (基于 slab/链表)
    │
    v
vmm_map()              — 虚拟内存映射
    │
    v
pmm_alloc()            — 物理内存分配 (位图)
    │
    v
E820 内存布局          — BIOS 探测物理内存
```

## 教程八：管道与进程间通信

### 8.1 创建和使用管道

```c
// 创建管道，返回文件描述符
int pipe_fd = pipe_create();
if (pipe_fd >= 0) {
    // 写入数据
    char msg[] = "Hello through pipe!";
    pipe_write(pipe_fd, msg, sizeof(msg));
    
    // 读取数据
    char buf[64];
    int nread = pipe_read(pipe_fd, buf, sizeof(buf));
    
    // 关闭管道
    pipe_close(pipe_fd);
}
```

### 8.2 管道阻塞模型

- 读空管道：读进程阻塞在 `read_waiters` 队列
- 写满管道：写进程阻塞在 `write_waiters` 队列
- 条件满足时自动唤醒等待进程

### 8.3 消息队列

```c
// 创建消息队列
int qid = Harlin_MsgCreate();

// 发送消息
Harlin_MsgSend(qid, data, size);

// 接收消息
Harlin_MsgRecv(qid, buf, &size);
```

### 8.4 信号量

```c
// 创建信号量（初始值 = 1）
int sem = Harlin_SemCreate(1);

// 等待（P 操作）
Harlin_SemWait(sem);

// 发送（V 操作）
Harlin_SemPost(sem);
```

## 教程十：常见问题排查

### 10.1 构建错误

**问题：`nasm: error: unable to open include file`**

检查 `%include` 路径是否相对于项目根目录。汇编中的 `%include` 使用相对于当前源文件的路径。

**问题：`undefined symbol`**

检查是否遗漏了某源文件的编译，或者链接脚本中未包含对应段。

**问题：`ld.lld: section .text lma overlaps`**

检查链接脚本中各段的 VMA/LMA 是否冲突。确保 `kernel.bin` 的加载地址与 stage2 中设置的加载地址一致。

### 10.2 运行错误

**问题：Bochs 启动后黑屏**

- 检查是否按了 `c` 键继续执行
- 检查 `bochsrc.txt` 中的镜像路径是否正确
- 检查 `BIOS-bochs-latest` 和 `VGABIOS-lgpl-latest.bin` 是否存在

**问题：内核启动后显示 "No bootable device"**

- 检查 `HarLin.iso` 是否正确生成
- 检查 `bochsrc.txt` 中 CD-ROM 配置是否正确
- 检查 boot.asm 中的 stage2 加载地址是否与 stage2 入口地址一致

**问题：系统调用返回错误值**

- 检查系统调用编号是否与 `syscall_table` 数组索引一致
- 检查参数传递约定（rdi/rsi/rdx）
- 检查 `int 0x80` 的段选择子是否正确（CS = 0x2B？）

### 10.3 调试技巧汇总

| 症状 | 检查方法 |
|------|----------|
| 内核完全无输出 | 检查 0xE9 调试端口输出 |
| 启动后重启（triple fault） | 使用 `reset_on_triple_fault=0` 观察错误位置 |
| 页表异常 | 检查 CR2 寄存器（出错地址） |
| 中断未触发 | 检查 IDT 条目和 PIC 重映射 |
| 多核 AP 未启动 | 检查 trampoline 代码和 SIPI 发送时序 |
| 网络不通 | 检查 RTL8139 IO 基址和 PCI 配置空间 |
