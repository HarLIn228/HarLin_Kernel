# HarLin Kernel 默认硬件指南


## 一、设计原则

- 每个子系统定义 ops 接口，驱动实现后通过 `drv_register` 注册
- 驱动可独立编译为模块，缺位硬件时由 NULL stub 提供 no-op 行为
- 内核仍可启动运行，只是不具备该子系统的高级能力

## 二、已支持的硬件标准与内置驱动

### 2.1 处理器与安全

| 组件 | 标准 | 内置实现 | 说明 |
|------|------|---------|------|
| CPU 特性探测 | x86_64，CPUID 枚举 SMEP/SMAP/PCID/FSGSBASE/RDRAND/RDSEED/NX/x2APIC | `feature_probe` | 启动阶段自动探测并启用对应 CR4 位 |
| 安全 | CR4.SMEP / CR4.SMAP / CR4.PCID | `cr4_features` | 基于 CPUID 结果动态使能 |
| 用户态隔离 | 内核/用户态隔离，用户指针校验 | `Harlin_UserPtrValid()` + `copy_from_user_safe()` / `copy_to_user_safe()` | 所有用户态数据拷贝必须经过安全封装 |

### 2.2 中断与计时

| 组件 | 标准 | 内置实现 | 说明 |
|------|------|---------|------|
| 中断控制器 | 8259A PIC（主片 IRQ0-7，从片 IRQ8-15） | `pic.asm` | 主片 IMR 屏蔽 IRQ5/6/7 |
| Local APIC | 多核 SMP 启动、IPI 发送 | `smp.c` + `lapic` | 支持 AP 引导 |
| 计时器 | PIT (8254) | `timer.asm` | 系统时钟基准 |

### 2.3 存储

| 组件 | 标准 | 内置实现 | 说明 |
|------|------|---------|------|
| 磁盘 | ATA PIO LBA28 | `ata.c` | LBA28 读写 |
| 分区 | MBR 分区表 | `partition.c` | 解析 MBR |
| 文件系统 | FAT32 | `fat32.c` | 文件读写、创建、删除、目录遍历 |

### 2.4 网络

| 组件 | 标准 | 内置实现 | 说明 |
|------|------|---------|------|
| 网卡 | RTL8139 (PCI 10/100M) | `drv_rtl8139` | PCI 网卡驱动 |
| 协议栈 | ARP / IP / ICMP / UDP / TCP / DHCP / DNS / HTTP 1.1 | `network.c` | DHCP 四步握手、DNS 域名解析、HTTP Content-Length + Chunked 传输编码 |

### 2.5 USB

| 组件 | 标准 | 内置实现 | 说明 |
|------|------|---------|------|
| 主机控制器 | UHCI (USB 1.1) | `usb.c` | 控制传输框架，含 data_len 边界校验 |

### 2.6 显示

|------|------|---------|------|
| VGA 文本模式 | 80x25 | `display.c` | 文本模式输出 |
| VGA 图形模式 | 320x200 256 色 (13h) | `screen.c` | 图形 Shell 渲染 |
| VESA 图形模式 | 800x600x24/32 | `screen.c` | 8x16 字体渲染 |

### 2.7 音频

## 三、引导路径

默认构建使用 **BIOS + MBR** 引导方式。

## 四、安全默认

- **CR4.SMEP** 已默认使能
- **CR4.SMAP** 已默认使能
- **CR4.PCID** 已默认使能
- 用户指针校验统一通过 `Harlin_UserPtrValid()` 入口
- 用户态拷贝统一通过 `copy_from_user_safe()` / `copy_to_user_safe()`

## 五、如何切换/裁剪驱动

编辑 `build.bat` 中驱动源文件列表，将不需要的 `.c` 移出 `src/harlin/drv/` 即可。内核启动时若某子系统无驱动，将通过 `drv_fallback_*` 打印提示并使用默认实现。

子系统的命名按 HarLin 习惯，例如 `DRV_SUBSYS_MEDIA`（存储）、`DRV_SUBSYS_LINK`（网络）、`DRV_SUBSYS_SCREEN`（显示）、`DRV_SUBSYS_SOUND`（声音）、`DRV_SUBSYS_CLOCK`（时钟）、`DRV_SUBSYS_BRIDGE`（总线）、`DRV_SUBSYS_GUARD`（看门狗），不照搬 Linux 命名。详见 `docs/第三方驱动接入.md`。
